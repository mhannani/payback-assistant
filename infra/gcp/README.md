# Deploy to GCP (Terraform)

The brief's preferred stack: **Cloud Run** (API) + **Cloud SQL** Postgres/pgvector (serving DB) +
**BigQuery** (the vector-search scale path, ADR 0003) + Artifact Registry + Secret Manager.

## What it provisions

| Resource | Role |
|---|---|
| Cloud Run service | Serves the FastAPI container on port 8000 |
| Cloud SQL (Postgres 16) | Serving DB — real-time `/search` + catalog rows, with pgvector |
| BigQuery dataset | Vector-search **scale** seam (ADR 0003); not the real-time store |
| Artifact Registry | Holds the API image |
| Secret Manager | Injects the LLM provider key at runtime, if any (never baked into the image) |
| Service account + IAM | Least-privilege runtime: read the secret, connect to Cloud SQL |

## Required deployer permissions

`terraform apply` enables the APIs and creates managed services, so the deploying principal needs
the rights to do so (the application's *runtime* permissions are the service account in the
Terraform; these are the *deployer's*). Grant the account running Terraform `roles/owner` on the
project, or — for least privilege — `roles/run.admin`, `roles/cloudsql.admin`,
`roles/artifactregistry.admin`, `roles/secretmanager.admin`, `roles/bigquery.admin`,
`roles/iam.serviceAccountAdmin`, and `roles/serviceusage.serviceUsageAdmin`.

## Runbook

```bash
cp terraform.tfvars.example terraform.tfvars     # fill in project_id / image
# The default all-Vertex path needs NO key (Cloud Run's service account authenticates via ADC),
# so nothing to export. For a keyed provider instead: export TF_VAR_llm_api_key=sk-... and set a
# matching llm_model (openai/… etc). The DB password is generated by Terraform.

make deploy-gcp      # terraform init + apply → prints the service URL  (needs gcloud ADC)
make seed-gcp        # one-off Cloud Run Job: init_db → seed → embed (populates Cloud SQL)
# → open the printed URL: /health, then POST /assist
make destroy-gcp     # tear it all down
```

The image defaults to the public GHCR image built by CI — nothing to build or push. (For an
all-GCP supply chain, push to the `image_repository` output and set `-var image=...`.)

> **Vertex AI:** swapping the LLM/embeddings to Vertex is a config change (`LLM_MODEL` →
> `vertex_ai/…`), not an infra change — LiteLLM routes to it. No Terraform edit required.

> **Cloud SQL connection:** Cloud SQL connects over a unix socket (`/cloudsql/<conn>`), not the
> `host:port` TCP form AWS/RDS uses. The app accepts an explicit `DATABASE_URL` override (config
> rewrites only the driver scheme, so the socket form `...@/db?host=/cloudsql/<conn>` round-trips
> intact for both the SQLAlchemy engine and the LangGraph checkpointer), and this module sets that
> socket `DATABASE_URL` on the Cloud Run service. The GCP module is no longer blocked on it.
